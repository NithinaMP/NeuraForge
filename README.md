## The Problem

Every product launch forces a marketing team to manually rewrite the same information вҖ” once for the blog, once for LinkedIn, once for email. The result is inconsistency, creative burnout, and content that slowly drifts from what the product actually does.

The deeper problem is not about writing speed. It's about **trust**. Single-model AI tools generate content confidently, but with no mechanism to verify what they wrote against what the source document actually said. A claimed feature that doesn't exist, a price point that was invented, a stat that was approximated вҖ” these slip through because nothing is checking.

Sentinel Assembly is built around a different principle: **content should be verified before it leaves the system.**

## Features

**Core Pipeline**
- Three-agent autonomous pipeline: Archivist вҶ’ Ghostwriter вҶ’ Prosecutor
- Self-correcting loop вҖ” runs until approved, up to 3 attempts
- Live agent log feed via Server-Sent Events (real-time, no polling)
- Audit Trail вҖ” visual timeline of every prosecution attempt and correction note

**Content Output**
- Blog post (~400 words, structured)
- Social media thread (5 posts, under 280 chars each)
- Email campaign (subject line + 90-word body with CTA)
- Mobile preview вҖ” see social posts rendered in a phone frame

**Agent Personality**
- Three selectable writing voices: Corporate Executive, Hype Beast, Minimalist
- Each personality dynamically injects a different system prompt into the Ghostwriter

**User System**
- JWT authentication вҖ” register, login, secure sessions
- Mission History вҖ” every run auto-saved to MySQL
- Mission Detail Page вҖ” 4 tabs: Campaign Results, Audit Trail, Fact Sheet, Source Document
- Export Kit вҖ” download full approved campaign as formatted text

**Technical**
- Server-Sent Events for real-time streaming (no WebSocket library needed)
- MySQL with auto-table creation on first run
- Graceful degradation вҖ” app works without MySQL (history disabled, pipeline still runs)

## Usage

1. Register an account or use the demo credentials button
2. Select a Ghostwriter personality (Corporate Executive / Hype Beast / Minimalist)
3. Paste a product document вҖ” or click **Load sample document** to test immediately
4. Click **Activate Sentinel Assembly**
5. Watch the three agents work in the live Agent Operations Room (left panel)
6. View the approved campaign вҖ” Blog, Social Thread, Email вҖ” in the right panel
7. Check the Audit Trail to see every prosecution loop and correction
8. Open Mission History to revisit any past run with full detail across 4 tabs

## Troubleshooting

| Problem | Solution |
|---|---|
| `OPENROUTER_API_KEY missing` | Check `server/.env` exists and key starts with `sk-or-v1-` |
| MySQL connection refused | Confirm MySQL is running; verify `DB_PASSWORD` in `.env` |
| `Port 3001 already in use` | Change `PORT=3002` in `server/.env` |
| Client shows proxy error | Server must be running before starting the client |
| Empty output sections | The free AI model may have hit rate limits вҖ” wait 30 seconds and retry |

> **Note:** The app runs without MySQL. If the database is unavailable, mission history is disabled but the AI pipeline works normally.
